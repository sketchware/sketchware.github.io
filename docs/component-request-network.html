Request Network

RequestNetwork is used to make Web API calls. You can acess Components tab.

To access it create a RequestNetwork component in the Components tab.

First, you need to understand some things about requests. The first "What is the component about?".

What is the component about?

Request Network component connects to a URL and retrieves data from it. The data can then used for our Application data.

Example

Using start network request to method to GET with the url URL with tag Tag.


Blocks

Set Headers

This block sets the headers for the request like user-agent.All these are defined in a map variable.

TypeExplanationRequired
RequestNetworkRequest Network ComponentYes
MapContains the headers in a map variableYes

Set Params

This block sets the request parameters. Request parameter starts with a question mark (?) . Web server reads the Request parameter from the URL and sends a Response based on the Request Parameter.

TypeExplanationRequired
RequestNetworkRequest Network ComponentYes
MapContains the parameters in a map variableYes
TypeThe type of the request.Yes

Start Network Request

Start the request to a specific URL .There are different types of method available.

MethodsExplanation
DELETEThe DELETE method deletes the specified resource.
GETThe GET method requests a representation of the specified resource. Requests using GET should only retrieve data.
POSTThe POST method is used to submit an entity to the specified resource, often causing a change in state or side effects on the server.
PUTThe PUT method replaces all current representations of the target resource with the request payload.

TypeExplanationRequired
RequestNetworkRequest Network ComponentYes
MethodsType of request.Yes
UrlThe hostname to request.Yes
TagThe internal name of the request.Yes
